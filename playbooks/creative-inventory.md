# Inventário Criativo e Diagnóstico Andromeda

## Quando usar

Use quando a conta tem queda de performance, CTR baixo, frequência alta, pouca escala, ou suspeita de que o problema não é só público/budget, mas falta de diversidade criativa.

## Princípio

O algoritmo precisa de variação real. Duplicar a mesma ideia com pequenas mudanças visuais não cria diversidade operacional suficiente.

## Entradas

- Lista de anúncios ativos e pausados relevantes.
- Criativo, formato, hook, ângulo, promessa, prova, CTA, público, objetivo.
- Métricas por anúncio: gasto, impressões, CTR, CPC, CPL, frequência, leads, reuniões, CPL real quando possível.
- Data de lançamento e última alteração.

## Classificação de diversidade

Um criativo é distinto quando muda pelo menos um eixo substantivo:

- Dor principal.
- Mecanismo/solução.
- Prova usada.
- Persona/segmento.
- Formato narrativo.
- Objeção atacada.
- CTA/oferta.
- Estágio do funil.

Não conta como criativo distinto:

- Troca pequena de cor.
- Crop diferente da mesma peça.
- Headline quase idêntica.
- Reupload sem nova hipótese.

## Thresholds operacionais

- `<5` criativos distintos: inventário pobre; leitura de performance é frágil.
- `5–9`: mínimo operacional para teste, mas limitado para escala.
- `10+`: base mais saudável para aprendizado e escala.
- `20+`: bom para contas com budget alto, desde que haja controle de hipóteses.

## Sinais de fadiga

- Frequência cold >3,5 com CTR caindo.
- CTR WoW caindo sem mudança de oferta.
- CPL sobe enquanto CPM fica estável.
- Mesma peça concentra gasto mas perde eficiência incremental.
- Comentários/rejeição aumentam.

## Matriz de inventário

Para cada criativo, registrar:

| Campo | Descrição |
|---|---|
| ID/nome | Sem expor dado sensível |
| Campanha/adset | Naming sanitizado |
| Formato | Estático, vídeo, carrossel, UGC, demo |
| Hook | Primeiros 3 segundos/primeira linha |
| Dor | Dor operacional específica |
| Mecanismo | Como a solução promete resolver |
| Prova | Case, número, autoridade, demonstração |
| Persona | ICP/persona atacada |
| CTA | Ação + benefício |
| Métrica líder | CTR/hook/hold/CPL |
| Métrica real | Lead Real/reunião/CPL real quando houver |
| Hipótese | Por que deveria funcionar |
| Decisão | Manter, iterar, pausar, remake |

## Decisão

- `Escalar`: bom desempenho em Meta e validação mínima no CRM.
- `Iterar`: bom sinal de atenção, mas conversão real fraca ou incompleta.
- `Pausar`: gasto suficiente + baixa atenção + baixo resultado real.
- `Remake`: bom conceito, execução fraca.
- `Investigar`: conflito entre Meta e CRM.

## Saída obrigatória

- Total de criativos distintos.
- Cobertura por dor/persona/formato.
- 3 lacunas de inventário mais importantes.
- 3 remakes prioritários.
- Risco de fadiga.
- Confiança da leitura.
