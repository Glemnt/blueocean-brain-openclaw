# History

Evidência histórica curada, sanitizada e datada.

## Papel

`history/` preserva aprendizados de relatórios, snapshots, decisões e linhagem antiga que são úteis para entender o desenho do sistema, mas não devem virar dado bruto versionado.

## O que entra aqui

- resumos datados sem PII;
- decisões de arquitetura;
- aprendizados históricos de campanhas, stack e operação;
- evolução de prompts/playbooks quando isso explicar uma decisão atual;
- evidência que ajuda a explicar por que o canon atual existe.

## O que não entra aqui

- CSV/TSV/export bruto;
- nomes, emails, telefones, documentos ou listas de clientes/leads;
- tokens, payloads integrais, prints sensíveis;
- relatórios HTML/PDF crus quando bastar resumo sanitizado.

## Regra de uso

Usar como contexto histórico, não como fonte atual de performance. Para decisão operacional atual, consultar fontes vivas e classificar confiança.
